# ADRAGON Design Context

## Scene

A technical reader opens the site after browsing a GitHub profile or a clean engineering blog. They are in normal daylight, scanning calmly, and want to know whether the builder has taste, persistence, and real systems behind the claims.

## Color Strategy

Restrained light interface with a warm paper base, charcoal text, quiet blue links, and small green or amber status signals. The site should feel closer to a well-maintained engineering notebook than a SaaS landing page.

Tokens:

- `--paper`: `oklch(0.985 0.006 90)`
- `--surface`: `oklch(0.965 0.007 90)`
- `--surface-2`: `oklch(0.94 0.01 95)`
- `--ink`: `oklch(0.205 0.015 250)`
- `--muted`: `oklch(0.48 0.02 250)`
- `--faint`: `oklch(0.68 0.018 250)`
- `--line`: `oklch(0.86 0.012 95)`
- `--blue`: `oklch(0.46 0.13 250)`
- `--green`: `oklch(0.45 0.11 155)`
- `--amber`: `oklch(0.55 0.12 65)`

## Typography

Use a system stack with confident but human proportions. Avoid giant theatrical headings. Body text and project descriptions should be the hero. Chinese text gets relaxed line-height and no forced negative tracking.

## Layout

The homepage uses a quiet personal-site composition inspired by clean engineering portfolios:

- Hero: narrow text column, direct identity, minimal actions.
- Artifact: Solo AI Company OS image shown as evidence inside a simple figure.
- Projects: editorial case index with one featured public project and four concise previews.
- Principles: short prose rows, not card decoration.
- Contact: direct text links.

## Interaction

Motion is nearly invisible. Hover states are link-like and quiet. Buttons should look like ordinary, trustworthy controls.
